# Gravity - 一站式自动化营销运营平台

> 面向企业的全自动运营中台：统一承接获客、培育、转化、复购、留存、召回和裂变等运营工作。

## 项目定位

Gravity 不是单点营销工具，也不是独立 CRM，而是一个可持续运行的运营系统。它通过统一的数据采集、身份识别、智能分群、策略决策、工作流编排、渠道执行和效果回流，把原本依赖人工的运营动作尽可能交给系统自动完成。

## 我们要解决什么

- 运营动作分散在多个工具里，流程断裂、数据割裂、责任不清
- 内容、投放、线索、跟进、成交、复购之间缺少统一闭环
- 大量重复操作依赖人工执行，成本高、错误率高、难以复制
- 传统自动化往往只覆盖局部，难以持续优化

## 核心能力

- **统一客户与线索中心**：汇聚多渠道用户数据，建立统一身份、标签、分群和行为画像
- **自动化工作流引擎**：触发、分支、等待、升级、回退、重试、补偿可编排
- **内容与话术工厂**：自动生成和改写文案、私信、邮件、短信、社群话术和活动素材
- **全渠道触达执行**：统一调度微信、企微、邮件、短信、内容平台、广告和落地页
- **智能转化编排**：自动判断触达时机、内容版本和下一步动作
- **复购与留存自动化**：自动召回沉默用户、唤醒老客、推动复购和会员运营
- **归因与实验系统**：支持漏斗、归因、A/B 测试、策略对照和效果分析
- **风控与审批治理**：高风险动作保留审批，低风险动作默认自动执行
- **企业级权限与审计**：支持组织、角色、审计日志、多租户隔离和安全控制

## 设计原则

1. **闭环优先**：每一次触达都必须形成反馈，反馈再驱动下一轮动作
2. **自动优先**：规则明确、风险可控、重复度高的工作优先自动化
3. **统一编排**：所有渠道、任务、素材、实验和回传都进入统一体系
4. **可配置优先**：不同业务、行业和阶段通过配置与模板适配
5. **可审计优先**：所有关键动作都能追踪、归因、复盘和审计

## 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | TypeScript + React + Vite |
| 后端 | Rust (Axum) |
| 数据库 | PostgreSQL + ClickHouse + Redis |
| 消息队列 | NATS |
| 部署 | Docker Compose / Kubernetes |

## 运行形态

- **SaaS**：适合标准化交付、多租户管理和快速上线
- **私有化部署**：适合对数据安全、网络边界和定制化要求更高的企业
- **双模式兼容**：同一套核心能力同时支持 SaaS 与私有化

## 快速开始

```bash
# 启动本地开发环境
docker compose up -d

# 后端开发
cd backend && cargo run

# 前端开发
cd frontend && npm install && npm run dev
```
